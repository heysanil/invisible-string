# Models

Rather than pick a raw model per workflow, you choose a **preset**. Each
workspace defines three — `powerful`, `balanced`, and `quick` — and the Agent
pillar selects among them. Presets are resolved against the workspace's
provider and model **allowlist**, so a workflow can only ever reference a model
the workspace has sanctioned.

Presets, the allowlist, and agent presets are seeded when a workspace is created
and managed under Settings. Model routing itself is baked into the compiled
artifact at build time, giving each published version a stable, pinned model
configuration.
