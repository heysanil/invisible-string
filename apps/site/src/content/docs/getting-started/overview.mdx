# Overview

invisible-string puts agents to work for you: give an agent a role, a
model, and the tools it needs, then chat with it like a teammate — or put it on
standing duty from Slack, forms, webhooks, or a schedule. Runs stream live,
pause for your approval, and are yours to run wherever you host them.

invisible-string is a multi-tenant cloud platform for AI agents. You build an
**Agent** — a persona, a model, and the context it's equipped with — and every
published Agent compiles into a real, self-hosted [eve](https://eve.dev) agent
that runs on a durable, Postgres-backed worker pool.

## Agents

An Agent is a role you define, fully described by three decisions:

### Persona

Who it is and how it works — the natural-language description of its role,
judgment, and working style.

### Model

How it thinks: `powerful`, `balanced`, or `quick`, resolved within the
workspace's model allowlist.

### Context

What it's equipped with: MCP connections and authored skills, packaged straight
into the compiled agent.

An Agent isn't a prompt template. Publishing compiles its definition into a
standalone, version-pinned artifact — built once, cached by content hash, and
run on a stateless worker pool where runs survive a worker dying mid-flight and
resume on any live worker. You chat with published Agents directly.

## Workflows

For standing work, delegate with a **workflow**: a trigger (webhook, form,
Slack event, or schedule), the published Agent that handles it, and
instructions for what to do when it fires, with `@trigger` references into the
event's data.

A workflow builds nothing — publishing is instant, and when the trigger fires
the platform delivers the event *and the instructions* to the bound Agent as
its task.
